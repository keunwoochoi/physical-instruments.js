- **2026-07-11 — Piano v2 + BS.1770/pyloudnorm loudness normalization (Keunwoo: "piano sucks", "use pyloudness").**
  Piano v2 fixes the three classic KS-piano failures: (1) pluck-like attack → hammer excitation = raised-cosine displacement pulse (width = fraction of string period, velocity-narrowed) blended with velocity-brightened noise; (2) single-stage decay → two-stage via string-pair asymmetry (dark long sustain string fed darker excitation; bright short bloom strings, bloom length velocity-scaled) per Weinreich polarization/coupling; (3) airless → 3-mode body-knock bank (85/172/318 Hz). Dispersion retuned down in bass ("watery"). Steering tool: `piano-audition.mjs` (two-stage t60 ratio, Hann-windowed spectral-centroid trajectory + audition WAV). **Measurement-tool lesson: the first centroid metric (unwindowed, coarse grid) reported RISING centroids that a peak analysis disproved (and a decimated debug DFT manufactured 23 kHz mirror ghosts) — instrument the instruments with the same rigor as the DSP.** Post-fix: two-stage ratios 3–7 at mf/ff, centroid falls bright→mellow, velocity→brightness strong (C2 307→774 Hz).
  Loudness: RMS proxy replaced with **pyloudnorm (BS.1770 integrated LUFS)** — `measure-loudness.mjs --wav-dir` + `measure-loudness.py`. K-weighting corrected the RMS lies: piano was +4.4 LU hot (user's ear right, twice), glock needed 4.7× MORE than the RMS-guess. Post-bake: **all 10 families at −26.2 LUFS ±0.1**.
  Verification: 14/14 cargo, LUFS table flat, demo 1.03% budget, E2E PASS 0 errors. Human listening: piano-audition.wav + instruments-demo.wav in scratchpad.
