# Docs site — scaffold

VitePress site, built out for the Q2 v0.1 release. Requirements fixed by the adoption research (approved plan):
- Landing page is the instrument: big Play button → beautiful recognizable note within ~1s of first tap, on mobile
- A draggable physical control (pluck position / tension / material) that audibly responds
- "< N KB, no samples" badge with the honest number (WASM included)
- Inline runnable snippets (Strudel/Glicol style) + StackBlitz open-in-editor buttons
- Examples gallery as a first-class product (the Three.js lesson)
